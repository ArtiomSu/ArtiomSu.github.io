What is QMK?

QMK is opensource firmware commonly used on keyboards and is very popular in the build your own keyboard scene as it can be flashed to inexpensive microcontrollers such as teensy and micro pro.

Compared to traditional or even high end keyboards from brands like logitech and corsair QMK allows you to do way more customisation, in other words it is much more powerful with its unique features.

Also compared to other keyboards that have advanced features, those advanced features require you to have the oem software running and drivers installed. With qmk this is not required everything is contained within the hardware the pc simply sees a normal keyboard and uses default drivers. This way you can setup your layouts and use them on windows, mac, linux, android without any setup whatsoever.

You can customise the firmware to your liking and its easy to add your own features though c code like I have done ( you can see all the features in for my custom layout below ). If you want to get started with compiling you can head over here But if you are scared of code you can use a number of graphical web configurators to change your layout.

Why should you use it?

It is very handy if you are a software developer or if you use a linux os with a window manager. In my case I use Arch Linux with i3 window manager. Window managers are heavily keyboard shortcut based and with qmk instead of pressing 5 buttons for a shortcut you can do it with 2 or even one. For example to launch firefox I use CTRL + Shift + Alt + Win + f if you tried doing that on a normal keyboard you would probably break your fingers but with qmk I just press 2 keys, one key is assigned to CTRL + Shift + Alt + Win and then I just tap f while holding that.

Another amazing aspect of qmk is the idea of layers. This is a little bit hard to explain without a video but it is essentially like your shift key but instead of pressing say shift a to make a capital A, you can program the firmware to output *, type some text, toggle leds, move your mouse etc....

Take a look at my layers in the layers tab for more info and explanation of what each layer does.

Ergodox EZ

My layout is built for the Ergodox Ez keyboard. You can see how it looks like at the top of the page.

It is a split keyboard so its perfect for ergonomics and allows you to stretch your hands out wide for a more comfortable experience.

If you are planning on buying one but have been using a traditional keyboard all your life it will take awhile to adjust to the the ortholinear layout but it is worth it trust me. It took me about 4 months to get used to it properly. In the first month I just used it occasionally as any real work would be too slow to do on it. Now I can actually type faster than I ever could on a regular keyboard, this is partly because you are forced to use one had per keyboard side therefore you wouldn't be pressing u,i etc with your left hand and so on, so it teaches you good typing habits whether you like it or not.

Features

Here I will go through most of the features specific to my layout for the ergodox again you can see a visual representation of the layout here

LEDs brightness

Increase/Decrease the keyboards backlight leds through Layer 3 annotated by led + and led - in the layout

Hue

Change the hue across all layers this pretty much rotates the colour wheel if you are familiar with image/video editors. Once again this is in Layer 3 and annotated by hue - and hue +

Toggle main layer leds

Allows you to toggle the leds on and off for the first layer only. This is in Layer 3 and annotated by Toggle layer 1 led

Toggle all LEDs

This is part of a leader key sequence go Here for a detailed explanation on how leader key sequences work. Essentially it lets you toggle the leds on and off after completing the sequence

Timeout

The keyboard will switch of its LEDs after 5 minutes of not pressing space. (space in my setup is essentially a macro as it is also part of the bunnyhop feature). Pressing space will turn on LEDs.

Bunnyhop

Bunnyhop is a term used in some games where you can gain speed by jumping repeatedly. The keyboard supports bunnyhopping with random delays so games dont detect it. To Enable bunny hopping press the toggle Bunnyhop button in layer 3. You will know when bunnyhopping is enabled because I programmed the = key to blink red when bunnyhop is active.

Leader Key Notification

This lets you know the time period for when the leader key is active as the bottom 3 switches will light up blue. For more information on leader key you can click here.

Uses four layer

See Layers more info.

Each layer has its own unique LED layout also. This allows you to easily distinguish between layers provided the LEDs are enabled.

Macros

A lot of macros are used for vim, ssh and so on you can find them in layers and Leader sections.

Custom LED Animation

These are present in my artiom_no_mouse_leds keymap which is slighly different to my default keymap artiom_no_mouse. If you are wondering what the no_mouse is for, its because I disabled the mouse functionality within the firmware to save space and also because I dont use it.
